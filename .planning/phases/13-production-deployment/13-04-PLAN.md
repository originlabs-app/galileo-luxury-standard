<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Configure custom domain and DNS</name>
  <action>
User must configure the galileoprotocol.io domain in Vercel dashboard and DNS:

**Step 1: Add domain in Vercel**
1. Go to https://vercel.com/dashboard
2. Select the galileo-protocol project
3. Go to Settings -> Domains
4. Click "Add Domain"
5. Enter: galileoprotocol.io
6. Also add: www.galileoprotocol.io (redirect to apex)

**Step 2: Configure DNS at your registrar**
Option A - A Record (recommended for apex domain):
- Type: A
- Name: @ (or galileoprotocol.io)
- Value: 76.76.21.21

CNAME for www:
- Type: CNAME
- Name: www
- Value: cname.vercel-dns.com

Option B - Vercel Nameservers:
- Update nameservers at registrar to:
  - ns1.vercel-dns.com
  - ns2.vercel-dns.com

**Step 3: Wait for DNS propagation**
- Can take up to 24-48 hours
- Check with: https://whatsmydns.net/#A/galileoprotocol.io
- Vercel will auto-provision SSL certificate once DNS is verified
  </action>
  <instructions>
After completing DNS configuration:
1. Wait for "Valid Configuration" status in Vercel Domains settings
2. Verify HTTPS is working: visit https://galileoprotocol.io
3. Check SSL certificate is valid (click padlock in browser)
  </instructions>
  <resume-signal>Type "domain configured" when galileoprotocol.io is live with HTTPS</resume-signal>
</task>

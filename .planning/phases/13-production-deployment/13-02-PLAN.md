Create directory .github/workflows/ if not exists.

Create .github/workflows/ci.yml:
```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: website/package-lock.json

      - name: Install dependencies
        run: npm ci
        working-directory: website

      - name: Run ESLint
        run: npm run lint
        working-directory: website

      - name: Build (includes TypeScript check)
        run: npm run build
        working-directory: website
```

Key points:
- Triggers on push to main and PRs
- Uses Node 20 (LTS)
- Caches npm dependencies
- Runs in website/ subdirectory
- ESLint and build (TypeScript) are separate steps for clear failure reporting

Create .github/workflows/lighthouse.yml:
```yaml
name: Lighthouse CI

on:
  pull_request:
    branches: [main]

jobs:
  lighthouse:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Wait for Vercel Preview
        uses: patrickedqvist/wait-for-vercel-preview@v1.3.1
        id: vercel-preview
        with:
          token: ${{ secrets.GITHUB_TOKEN }}
          max_timeout: 300

      - name: Audit URLs with Lighthouse
        uses: treosh/lighthouse-ci-action@v12
        with:
          urls: |
            ${{ steps.vercel-preview.outputs.url }}
            ${{ steps.vercel-preview.outputs.url }}/docs
            ${{ steps.vercel-preview.outputs.url }}/blog
          budgetPath: ./website/lighthouse-budget.json
          uploadArtifacts: true
          temporaryPublicStorage: true
```

Create website/lighthouse-budget.json:
```json
{
  "ci": {
    "assert": {
      "assertions": {
        "categories:performance": ["error", { "minScore": 0.9 }],
        "categories:accessibility": ["error", { "minScore": 0.9 }],
        "categories:best-practices": ["error", { "minScore": 0.9 }],
        "categories:seo": ["error", { "minScore": 0.9 }]
      }
    }
  }
}
```

Key points:
- Lighthouse CI runs only on PRs (preview deployments)
- Waits for Vercel preview to be ready (up to 5 min)
- Tests 3 key pages: home, docs, blog
- Budget enforces >90 (0.9) on all 4 categories
- Uploads artifacts for debugging

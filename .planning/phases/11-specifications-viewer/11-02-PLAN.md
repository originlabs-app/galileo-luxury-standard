Create `website/src/app/specifications/[category]/[slug]/page.tsx`:

**Route:** Dynamic route for individual spec pages

**Required exports:**
1. `generateStaticParams()` - pre-render all spec pages
2. `generateMetadata()` - SEO metadata
3. `default` - page component (Server Component)

**generateStaticParams implementation:**
```typescript
export async function generateStaticParams() {
  const categories = await getSpecCategories();
  const params = [];

  for (const category of categories) {
    const specs = await getSpecifications(category);
    for (const spec of specs) {
      params.push({ category, slug: spec.slug });
    }
  }

  return params;
}
```

**Page component logic:**
1. Call `getSpecification(params.category, params.slug)`
2. If type === 'markdown':
   - Serialize content with `next-mdx-remote/serialize`
   - Render with `<MDXRemote>` component (client component)
   - Pass CodeBlock component for syntax highlighting (already exists)
3. If type === 'json':
   - Render with `<JSONSchemaViewer schema={content} />`
4. Display `<SpecMetadata>` above content
5. Wrap content in `prose prose-invert max-w-none` for typography

**MDX rendering:**
```tsx
import { serialize } from 'next-mdx-remote/serialize';
import { MDXRemote } from 'next-mdx-remote/rsc';
import { CodeBlock } from '@/components/ui/CodeBlock';

// In page component:
const mdxSource = await serialize(spec.content, {
  mdxOptions: {
    remarkPlugins: [],
    rehypePlugins: [],
  },
});

// Render:
<div className="prose prose-invert max-w-none">
  <MDXRemote {...mdxSource} components={{ pre: CodeBlock }} />
</div>
```

**generateMetadata:**
```typescript
export async function generateMetadata({ params }: { params: { category: string; slug: string } }) {
  const spec = await getSpecification(params.category, params.slug);
  return {
    title: `${spec.metadata.title} | Galileo Specifications`,
    description: `${spec.metadata.title} - Version ${spec.metadata.version} (${spec.metadata.status})`,
  };
}
```

**Why use next-mdx-remote:** Allows rendering markdown from any source (not just .mdx files). Supports async data fetching in Server Components. Already installed, no new dependencies.

**Why separate JSON viewer:** JSON schemas need different rendering than markdown. Viewer component can be enhanced later with interactive features (collapsible sections, type highlighting).

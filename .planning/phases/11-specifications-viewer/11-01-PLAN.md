  <files>website/src/lib/specifications.ts</files>
  <action>
Create `website/src/lib/specifications.ts` with functions for reading and parsing specification files:

**Key functions:**
- `getSpecCategories()`: Read ../specifications/ directory, return category names (architecture, compliance, contracts, crypto, identity, infrastructure, resolver, schemas, token)
- `getSpecifications(category)`: Read all .md and .json files in a category, parse metadata, return spec objects with slug, title, status, version, lastUpdated
- `getSpecification(category, slug)`: Read single spec file (try .md first, then .json), return content and metadata
- `parseMarkdownMetadata(content)`: Parse bold markdown headers (NOT YAML frontmatter) - extract Status, Version, Last Updated, Specification Series from first 20 lines

**Important implementation details:**
- Use `path.join(process.cwd(), '../specifications')` for absolute path from website/ to specifications/
- Use `fs.promises` for async file operations
- For metadata parsing: Use regex to find `**Status:**`, `**Version:**`, `**Last Updated:**`, `**Specification Series:**` (or `**Specification ID:**`)
- Extract title from first `# Heading` line
- Return type: `{ slug, filename, category, type: 'markdown' | 'json', title, status: 'Draft' | 'Active' | 'Standard', version, lastUpdated, specId? }`
- Sort specs alphabetically by title within each category
- For JSON files: Parse JSON, extract title from schema's `title` field, status defaults to 'Standard'

**Why parse custom metadata format:** Galileo specs use bold markdown, not YAML frontmatter. Converting all 42 files would be error-prone. Parser must handle existing format.

**TypeScript interfaces:**
```typescript
interface SpecMetadata {
  title: string;
  status: 'Draft' | 'Active' | 'Standard';
  version: string;
  lastUpdated: string;
  specId?: string;
}

interface SpecFile {
  slug: string;
  filename: string;
  category: string;
  type: 'markdown' | 'json';
  title: string;
  status: 'Draft' | 'Active' | 'Standard';
  version: string;
  lastUpdated: string;
  specId?: string;
}
```
